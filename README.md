# gt_api

谷通公用类API

## 说明
#### com.gt.api 代码包
- 按照规则命名，不要写重复的util及功能
- 要求所有的类和方法都要写清楚注释
- 由项目负责人需review提交的代码，对不合格的地方应指出及要求对应人员修改
#### 包说明
- util包
    - 编写工具类
    - 类目命名规则：XxxUtils
        
#### test.com.gt.api 测试包
- 按照规则命名
- 所有类及方法都应有对应的测试类
- 对测试代码覆盖率进行检测，要求不低于50%
#### 包说明
- util包
    - 对应代码包的util
    - 类命名规则：XxxUtilsTest
    - 方法命名规则：xxxxTest
